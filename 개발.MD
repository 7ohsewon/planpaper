# 기안문 작성기 개발 기록

## 개요

- 목표: 업로드한 문서에서 텍스트를 추출해 AI로 기안문 초안을 만들고 `.hwpx` 파일로 다운로드하는 작성기 구현
- 실행 환경: Streamlit
- 지원 입력: `pdf`, `hwpx`, `docx`, `txt`
- 출력: `hwpx`

## 기존 구현 확인

- `extractor.py`
  - PDF, HWPX, DOCX, TXT에서 텍스트 추출
- `draft.py`
  - 원문 텍스트를 바탕으로 제목, 본문, 붙임이 포함된 JSON 초안 생성
- `hwpx_writer.py`
  - 템플릿 `기안문.hwpx`에 초안 내용을 주입해 새 HWPX 생성
- `app.py`
  - Streamlit UI에서 업로드, 추출, 초안 생성, 다운로드 흐름 제공

## 이번 작업 내용

### 1. LLM 연동 구조 확장

- 기존 `Anthropic` 전용 구현을 `OpenAI 호환`과 `Anthropic` 선택형으로 변경
- `app.py`에서 다음 입력을 받도록 수정
  - `LLM 제공자`
  - `API Access Token` 또는 `Anthropic API Key`
  - `모델`
  - `Base URL (선택)`

### 2. OpenAI Responses API 연동

- `draft.py`에 `openai` SDK 추가
- OpenAI 경로는 `client.responses.create(...)` 사용
- 기본 모델은 `gpt-5.5`로 설정
- JSON 응답만 파싱하도록 공통 처리 유지

### 3. 의존성 추가

- `requirements.txt`에 `openai>=1.0.0` 추가
- 실제 설치 완료

### 4. 인증 입력 안정화

- `OPENAI_API_KEY` 환경변수에 개행이 포함된 상태를 확인
- `app.py`, `draft.py`에서 API 키 입력값에 `strip()` 적용
- 이 수정으로 헤더 생성 오류 방지

## 실행 및 검증 기록

### 서버 실행

- Streamlit 서버를 `8511` 포트로 실행
- 현재 접속 주소: `http://localhost:8511`

### 문법 검증

- `python -m py_compile app.py draft.py extractor.py hwpx_writer.py`
- 결과: 통과

### OpenAI SDK 확인

- `openai` 패키지 설치 완료
- import 및 버전 확인 완료

### 네트워크/인증 확인

- `https://api.openai.com/v1/models` 도달 가능 확인
- 사용자가 제공한 `AQ...` 토큰은 OpenAI 공식 API 키가 아니어서 `401 invalid_api_key` 확인
- 환경변수의 `OPENAI_API_KEY`는 공식 키 형식(`sk-proj...`)이며 개행 제거 후 인증 성공 확인

### 실제 API 호출 확인

- 짧은 테스트 입력으로 `generate_draft(...)` 직접 호출
- 결과: 인증은 통과했지만 `429 insufficient_quota`

## 현재 상태

- 앱 실행: 정상
- 문서 추출: 정상
- HWPX 생성 로직: 정상
- OpenAI 인증: 정상
- 남은 문제: OpenAI 프로젝트의 사용 가능 quota 부족

## 결론

현재 기안문 작성기는 기능적으로 실행 가능한 상태다. 다만 OpenAI 공식 API를 통한 초안 생성은 프로젝트 quota가 부족해서 실제 생성 단계에서 `429 insufficient_quota`로 막힌다. 결제 또는 사용 한도 문제를 해결하면 같은 코드로 바로 동작시킬 수 있다.
